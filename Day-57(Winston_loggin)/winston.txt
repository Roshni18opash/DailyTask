										WINSTON
				
Winston is a popular open-source logging library for Node.js. It provides a flexible, transport-based system for capturing and managing log messages across various outputs. Widely used in server-side JavaScript applications, Winston emphasizes configurability, structured logging, and extensibility.

What is Winston?
Winston is a logging library for Node.js.
It helps you:
Save logs in files
Separate error logs
Add timestamps
Control log levels (info, error, warn)

Logging kya hota hai?
Logging ka matlab hai —
👉 App ke andar jo bhi important cheezein ho rahi hain (login, error, server start, API call), unka record rakhna.
Jaise CCTV camera record rakhta hai,
waise hi logging system backend ka record rakhta hai.

Problem with console.log()
console.log() sirf terminal me dikhata hai.
❌ File me save nahi karta
❌ Error alag se store nahi karta
❌ Time automatic nahi lagata
Production app me ye enough nahi hota.
Winston kya hai?

Winston ek Node.js logging library hai.
Ye:
✔ Logs ko file me save karta hai
✔ Error aur info alag karta hai
✔ Time stamp add karta hai
✔ Professional backend banata hai

structuire:
const logger = winston.createLogger({
   level: "info" //log ka type save krta he
});

Winston me different log types hote hain:

error ❌ (jab error aaye)

warn ⚠️ (warning)

info ℹ️ (normal information)

Winston me ek cheez hoti hai jise bolte hain transport.
Ye decide karta hai log kahan jayega (console ya file).

Transport ka kaam kya hoga?
Log kahan store hoga decide karna

Level → Kaunsa type ka log (info, error, warn)
Transport → Log kahan jayega (console ya file)

Matlab:

Console transport → Terminal me dikhayega

File transport → File me save karega

Add transport in winston below 
transports: [
   new winston.transports.Console(),
   new winston.transports.File({ filename: "app.log" })
]
